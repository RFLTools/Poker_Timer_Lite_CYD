# Web Flasher for Poker Timer CYD

This directory contains the web-based firmware flasher that allows users to install the Poker Timer firmware directly from their web browser without needing to install any software.

## Files

- `index.html` - Main web flasher interface
- `manifest.json` - ESP Web Tools manifest for firmware flashing
- Firmware binaries (generated by build process):
  - `firmware.bin` - Main application firmware
  - `bootloader.bin` - ESP32 bootloader
  - `partitions.bin` - Partition table
  - `boot_app0.bin` - Boot application

## How It Works

The web flasher uses [ESP Web Tools](https://esphome.github.io/esp-web-tools/) which leverages the Web Serial API to communicate with ESP32 devices directly from the browser.

### Supported Browsers

- Google Chrome (version 89+)
- Microsoft Edge (version 89+)
- Opera (version 75+)

**Note:** Firefox and Safari do not currently support the Web Serial API.

## Testing Locally

To test the web flasher locally:

1. Build the firmware first (see below)
2. Start a local web server in this directory:
   ```bash
   python -m http.server 8000
   ```
3. Open http://localhost:8000 in a supported browser
4. Connect your ESP32 device and click the install button

## Building Firmware

### Using PlatformIO CLI

```bash
# From project root
pio run --environment esp32-2432s028
```

### Using Visual Studio Code

1. Open project in VS Code with PlatformIO extension
2. Click "Build" in the PlatformIO toolbar
3. Or use Command Palette: `PlatformIO: Build`

### Copying Binaries

After building, copy the binaries to the web_flasher directory:

```bash
# Windows (PowerShell)
./build_web_flasher.ps1

# Linux/Mac
./build_web_flasher.sh
```

Or manually:
```bash
cp .pio/build/esp32-2432s028/firmware.bin web_flasher/
cp .pio/build/esp32-2432s028/bootloader.bin web_flasher/
cp .pio/build/esp32-2432s028/partitions.bin web_flasher/
```

## GitHub Pages Deployment

The web flasher is automatically deployed to GitHub Pages when you push to the main branch. The GitHub Actions workflow:

1. Builds the firmware using PlatformIO
2. Copies all necessary files to a distribution folder
3. Deploys to GitHub Pages

### Accessing the Web Flasher

Once deployed, the web flasher will be available at:
```
https://rfltools.github.io/Poker_Timer_Lite_CYD/
```

## Enabling GitHub Pages

1. Go to your repository settings
2. Navigate to "Pages" section
3. Under "Build and deployment":
   - Source: "Deploy from a branch"
   - Branch: "gh-pages"
   - Folder: "/ (root)"
4. Save the settings

The GitHub Actions workflow will handle the deployment automatically.

## Troubleshooting

### Port not appearing in browser

- Ensure you're using a supported browser (Chrome/Edge/Opera)
- Check that the USB driver (CP210x or CH340) is installed
- Try a different USB cable (some are power-only)
- Close any other programs using the serial port

### Installation fails

- Hold the BOOT button while connecting USB
- Try a different USB port
- Restart the browser
- Check the browser console for error messages

## Technical Details

### Flash Memory Layout

| Address | Size  | Content           |
|---------|-------|-------------------|
| 0x1000  | 24KB  | Bootloader        |
| 0x8000  | 3KB   | Partition Table   |
| 0xE000  | 8KB   | Boot App          |
| 0x10000 | ~1.5MB| Main Application  |

### Partition Scheme

The project uses a custom partition scheme (`no_ota.csv`) that maximizes application space by removing OTA partitions:

- app0: 1,600,000 bytes (~1.5MB)
- spiffs: 400,000 bytes (~400KB)

This allows for a larger application with more features.

## Security Considerations

The web flasher requires:
- Physical access to the device (USB connection)
- User interaction (selecting the port and confirming installation)
- HTTPS when hosted (GitHub Pages provides this automatically)

Users should only install firmware from trusted sources.
